## 1. Какая команда используется для настройки сетевых интерфейсов?

Для настройки сетевых интерфейсов используется команда `ip`
Она позволяет добавлять и удалять IP-адреса, включать и выключать интерфейсы, управлять маршрутами и туннелями

Примеры использования:

```bash
ip addr add 192.168.1.10/24 dev eth0     # назначить адрес
ip link set eth0 up                      # включить интерфейс
ip addr show                             # вывести текущие настройки
```

Также применялась команда `ifconfig`, но считается устаревшей и заменяется `ip`

## 2. Для чего используется команда route?

Команда `route` применяется для просмотра и изменения таблицы маршрутизации.

Примеры:

```bash
route -n                                # вывод таблицы маршрутизации
route add default gw 192.168.1.1        # задать основной шлюз
```

Современный аналоги - это использование подкоманды `ip route`:

```bash
ip route show
ip route add default via 192.168.1.1
```

## 3. Команды диагностики сети

Для проверки состояния сети и поиска неисправностей применяются следующие основные утилиты:

| Команда                          | Назначение                                            | Пример             |
| -------------------------------- | ----------------------------------------------------- | ------------------ |
| **ping**                         | Проверка доступности узла и времени отклика.          | `ping 8.8.8.8`     |
| **traceroute / tracepath**       | Определение маршрута следования пакета до цели.       | `traceroute yandex.ru` |
| **nslookup / dig / host**        | Проверка работы DNS, получение IP по доменному имени. | `dig example.com`  |
| **ss / netstat**                 | Просмотр активных соединений и открытых портов.       | `ss -tuln`         |
| **ip addr / ip link / ip route** | Вывод информации о сетевых интерфейсах и маршрутах.   | `ip route show`    |

## 4. Команды удалённого терминала

Для подключения к другим системам через сеть используются команды удалённого доступа:

`ssh` (Secure Shell) — безопасное шифрованное подключение к удалённому терминалу.

```bash
ssh user@192.168.1.5
```

Также через SSH можно копировать файлы с помощью `scp`.

## 5. Команда `iptables`

`iptables` — это средство управления встроенным межсетевым экраном (фаерволом) ядра Linux.
С его помощью можно фильтровать пакеты, запрещать или разрешать доступ к портам, выполнять перенаправление трафика и настраивать NAT.

Основные цепочки (chains):

* INPUT — обработка входящих пакетов;
* OUTPUT — исходящие пакеты;
* FORWARD — транзитные пакеты, проходящие через систему.

Пример настройки:

```bash
iptables -A INPUT -p tcp --dport 22 -j ACCEPT   # разрешить SSH
iptables -A INPUT -j DROP                       # запретить остальной входящий трафик
```

Ключевые параметры:

- `-A` — добавить правило;
- `-p` — указание протокола (tcp, udp, icmp);
- `--dport` — порт назначения;
- `-j` — действие (ACCEPT, DROP, REJECT и др.).
